### 1. Business Creation Pipeline Page (ZERO → HERO)  

**Core Purpose**  

A single Kanban-style + detail-view interface where admins (or eventually an “Open Claw” AI agent) guide customer businesses from raw lead → profitable autonomous operation.  



- **Global List View** (default)  

  - Table of all pipeline records with columns: Business Name, POC, Current Phase, Started, Last Activity, Revenue Potential, Status (Active / Suspended / Completed)  

  - Filters: Phase, POC Email, Referral Source, Admin Owner  

  - “New Project” button → opens Phase 00 creation wizard  



- **Per-Record Detail View** (click any row)  

  - Horizontal phase stepper (00–06) with progress indicators and “Advance Phase” button (with required checklist)  

  - Tabbed sections inside each phase (Notes, Artifacts, Voice Transcripts, AI Outputs, etc.)  

  - One customer can have multiple businesses at different phases (stored as separate `pipeline_projects` records linked to the same `user_id`).



**Database Additions Needed** (minimal, builds on existing Drizzle schema)  

Add to `drizzle/schema.ts`:

```sql

CREATE TABLE pipeline_projects (

  id SERIAL PRIMARY KEY,

  user_id INTEGER REFERENCES users(id),           -- customer

  admin_id INTEGER REFERENCES users(id),          -- GoGetter admin owner

  name VARCHAR(255) NOT NULL,

  poc_name VARCHAR(255) NOT NULL,

  poc_email VARCHAR(255),

  poc_phone VARCHAR(100),

  referral_source VARCHAR(100),                   -- ad, oauth, promo, etc.

  phase INTEGER DEFAULT 0 CHECK (phase BETWEEN 0 AND 6),

  status VARCHAR(50) DEFAULT 'active',

  started_at TIMESTAMP DEFAULT NOW(),

  metadata JSONB DEFAULT '{}',                    -- all artifacts, transcripts, prompts, etc.

  created_at TIMESTAMP DEFAULT NOW(),

  updated_at TIMESTAMP DEFAULT NOW()

);

```
